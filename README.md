# DESAFIO-QA-BEEDOO-2026
Este projeto foi criado como teste para a vaga Analista de Qualidade de Software Júnior.

## 1. Qual você acredita ser o objetivo da aplicação?
R| Essa aplicação tem como objetivo possibilitar o cadastro e gerenciamento de cursos na plataforma, possibilitando ao uusuário registrar novos cursos,
visuualizar os cuursos cadastrados e visualizar uma lista mais aprofundada dos cadastros realizados. O sistema funciona como um módulo simplificado de
gerenciamento e adição de cursos.

## 2. Quais são os principais fluxos disponíveis?
R| Fluxo 1 - Cadastro de Curso
  a. Usuário acessa a tela de cadastro
  b. Preenche os campos do formulário
  c. Clica em Salvar / Cadastrar
  d. O curso é armazenado no sistema
  e. O usuário é redirecionado para a lista de cursos

Fluxo 2 - Listagem de Cursos
  a. UUsuário acessa a tela de listagem
  b. Sistema apresenta todos os cursos cadastrados
  c. Usuário pode visualizar informações do curso

Fluxo 3 - Validação de Formulário
  a. Usuário tenta cadastrar curso
  b. Campos obrigatórios são verificados
  c. Sistema exibe mensagens de erro caso necessário

## 3. Quais pontos do sistema você considera mais críticos para teste?
R| * Validação dos campos obrigatórios
   * Persistência dos dados cadastrados
   * Integridade da listagem de cursos
   * Comportamento do sistema com dados inválidos
   * Comportamento do sistema ao cadastrar cursos duplicados
   * Usabilidade do formulário
   * Feedback visual ao usuário (erros e sucesso)
